# خريطة شواحن السيارات الكهربائية في سورية

تطبيق ويب تفاعلي لعرض شواحن السيارات الكهربائية في سورية، مع فلاتر، واجهة عربية/إنجليزية، ولوحة إدارة محمية مرتبطة بـFirebase وCloud Firestore.

## المزايا الحالية

- واجهة عربية RTL وإنجليزية LTR ومتوافقة مع الهاتف.
- خريطة تفاعلية باستخدام Leaflet وOpenStreetMap.
- فلاتر حسب المحافظة، المشغل، نوع المقبس، الحالة، ونوع الموقع.
- إحصاءات مباشرة لعدد المواقع والشواحن والمخارج والقدرة والمحافظات.
- بطاقات تفصيلية لكل موقع مع المصدر وجودة البيانات.
- لوحة إدارة خاصة غير مرتبطة أو ظاهرة في الواجهة العامة.
- تسجيل دخول الإدارة عبر Firebase Authentication بالبريد وكلمة المرور.
- صلاحيات كتابة محمية عبر Firebase ID Token وقائمة بريد إلكتروني مسموحة.
- تخزين البيانات في Cloud Firestore من خلال Vercel Functions فقط.
- إدخال وتعديل وحذف الشواحن مع سجل تدقيق إداري.
- استخراج الإحداثيات من روابط Google Maps، مع طلب Latitude وLongitude عند تعذر الاستخراج.
- استيراد وتصدير Excel وCSV وJSON.
- تحقق يمنع نشر إحداثيات خارج النطاق الجغرافي السوري المقبول.
- رجوع آمن إلى ملف JSON الحالي قبل اكتمال إعداد Firebase أو عند تعذر الوصول إليه.

## لوحة الإدارة

لا يوجد أي زر أو رابط عام للوحة الإدارة. بعد إعداد Firebase ونشر التطبيق، تُفتح مباشرة عبر:

```text
https://YOUR_DOMAIN/admin/
```

إخفاء الرابط ليس آلية الأمان الأساسية؛ الحماية الفعلية تعتمد على التحقق من Firebase ID Token، البريد الإداري المسموح، تحقق البريد الإلكتروني، وسياسات الوصول من جهة الخادم.

## حقول السجل الأساسية

| الحقل | الوصف |
|---|---|
| `Suggested_ID` | معرّف فريد للسجل |
| `Company_AR` / `Company_EN` | الشركة بالعربية والإنجليزية |
| `Governorate_AR` / `Governorate_EN` | المحافظة بالعربية والإنجليزية |
| `City_AR` / `City_EN` | المدينة بالعربية والإنجليزية |
| `Site_Name_AR` / `Site_Name_EN` | اسم الموقع بالعربية والإنجليزية |
| `Charger_Type_AR` / `Charger_Type_EN` | نوع الشاحن بالعربية والإنجليزية |
| `Charger_Numbers` | عدد الشواحن في الموقع |
| `Site_Type_AR` / `Site_Type_EN` | نوع الموقع بالعربية والإنجليزية |
| `Google_Maps_URL` | رابط Google Maps |
| `Latitude` / `Longitude` | إحداثيات تلقائية أو يدوية عند الحاجة |
| `Published` | منشور للعامة أو مسودة |

## مصدر بيانات الخريطة العامة

يبقى التطبيق العام يطلب المسار:

```text
/data/chargers.v2.json
```

لكن Vercel يعيد توجيهه داخلياً إلى:

```text
/api/chargers
```

وتقوم الواجهة البرمجية بقراءة السجلات المنشورة من Firestore. عند عدم إعداد Firebase أو عدم وجود سجلات منشورة، يمكنها استخدام ملف `data/chargers.v2.json` كنسخة احتياطية مؤقتة.

## إعداد Firebase وVercel

راجع الدليل التنفيذي الكامل:

```text
docs/FIREBASE_ADMIN_SETUP.md
```

يتضمن الدليل:

- إنشاء Firebase Web App.
- تفعيل Email/Password Authentication.
- إنشاء حساب الإدارة والتحقق من البريد.
- إنشاء Firestore ونشر القواعد المغلقة.
- إعداد Service Account.
- إضافة متغيرات البيئة إلى Vercel.
- استيراد البيانات الحالية واختبار الربط.
- قائمة تحقق أمنية وتشغيلية.

## متغيرات البيئة

استخدم الملف:

```text
.env.example
```

ولا ترفع مطلقاً ملف Service Account JSON أو المفتاح الخاص إلى GitHub.

- رفض الإحداثيات الواقعة خارج سورية.

## قيم الحالة وجودة البيانات

تبقى حالات التشغيل وجودة البيانات منفصلة عن حالة النشر. السجل المنشور لا يعني بالضرورة أنه موثق ميدانياً أو عامل؛ يجب استخدام حقلي `status` و`data_quality` وفق منهجية المنصة.

## النشر على Vercel

1. أضف متغيرات البيئة المطلوبة.
2. تأكد من إعداد Firebase Authentication وFirestore.
3. ادمج التغييرات في `main`.
4. نفّذ عملية نشر واحدة بعد اكتمال الإعداد.
5. اختبر `/admin/` ثم الخريطة العامة.
6. تحقق من أن `/api/chargers` يعرض `storage: firebase-firestore` بعد استيراد البيانات.
